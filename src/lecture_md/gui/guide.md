# lecture-md 使用说明

把「讲课声音 + PPT 画面」的课程录屏,自动整理成**按 PPT 页对齐的 Markdown 讲义**。

## 工作原理

每个视频会依次经过四个阶段:

1. **切页去重** — 检测翻页,提取每页 PPT 截图和时间轴;防抖算法过滤鼠标移动、临时翻回等造成的假切页。
2. **按页转写** — 按每页的时间范围切出音频,逐页做语音识别(本机 Whisper 或 API)。
3. **纠错优化** — OCR 读取 PPT 页面上的文字,大模型对照修正转写中的错别字、专业术语和断句。
4. **讲义生成** — 大模型把每页的逐字稿提炼成适合复习的讲义稿,公式转为 LaTeX。

所有中间结果增量写盘:中断、断网、限流后**重跑同一个任务即可续跑**,已完成的页面不会重做。

## 四种处理方案对比

| | 仅转写(免费) | 转写 + 纠错 | 完整讲义(推荐) | 全 API |
| --- | --- | --- | --- | --- |
| 切页去重 | ✓ | ✓ | ✓ | ✓ |
| 语音转写 | 本机 Whisper | 本机 Whisper | 本机 Whisper | **API** |
| 错别字/术语纠错 | ✗ | ✓ API | ✓ API | ✓ API |
| 讲义稿生成 | ✗ | ✗ | ✓ API | ✓ API |
| 最终产物 | 原始逐字稿 | 干净逐字稿 | **讲义稿**(三层内容) | **讲义稿**(三层内容) |
| 需要 API 密钥 | 不需要 | 需要 | 需要 | 需要 |
| 需要 faster-whisper | 需要 | 需要 | 需要 | 不需要 |
| Token 消耗 | 0 | 低 | 中 | 高(音频也走 API) |
| 本机算力消耗 | 高 | 高 | 高 | 几乎为零 |

几点说明:

- **「完整讲义」和「全 API」的产物完全相同**,区别只在语音转写是在你电脑上跑还是发给 API。电脑有不错的 CPU/GPU 就用「完整讲义」省 Token;电脑不想跑重活就用「全 API」。
- 「讲义稿(三层内容)」指每页同时保留:精炼讲义稿、校正后的逐字稿、ASR 原文,方便复习时溯源。
- 转写质量参考:本机 Whisper 的 `small` 模型对中文课程已经够用;追求更准可以在设置里换 `medium`,有 NVIDIA 显卡可把推理设备改为 CUDA,速度提升数倍。

## 准备工作

### 1. 环境检查(设置页右下角)

打开「设 置」页,环境检查会列出各依赖的状态,**未安装的直接点「安装」按钮即可**:

| 依赖 | 作用 | 什么时候需要 |
| --- | --- | --- |
| ffmpeg | 按页切分音频 | 所有方案必需(一键安装走 static-ffmpeg,自动下载,无需配 PATH) |
| slidegeist | 翻页检测 | 所有方案必需 |
| faster-whisper | 本机语音转写 | 除「全 API」外的所有方案 |
| RapidOCR | 读取 PPT 页面文字辅助纠错 | 用到「纠错」的方案 |

### 2. API 配置(设置页左上角)

凡是用到纠错/讲义/API 转写的方案,都需要一个 **OpenAI 兼容**的 API:

- **API 地址**:下拉框内置 OpenAI、DeepSeek、MiMo、Ollama(本机)预设,也可手动输入。
- **API 密钥**:填好后点「测试连接」验证;留空则使用环境变量 `LECTURE_MD_API_KEY`。
- **文本模型**:用于纠错和讲义,例如 `mimo-v2.5-pro`、`deepseek-chat`、`gpt-4o-mini`。
- **音频模型**:仅「全 API」方案需要,必须是支持 `input_audio` 的对话模型,例如 `mimo-v2.5-asr`、`gpt-4o-audio-preview`。
- **术语表**:把课程的专业词汇用逗号列出(如 `流水线, 数据冒险, Cache 一致性`),纠错准确率会明显提升。

所有设置自动保存,下次启动直接生效。

## 处理中心

1. 把视频文件或**整个文件夹**拖进虚线区域(或点击选择),任务进入队列;
2. 右上角选择处理方案,下方有一行说明告诉你该方案的产物;
3. 点「开始处理」,队列按顺序逐个执行;
4. 右侧实时显示当前阶段和「第 x/yy 页」进度,勾选「显示运行日志」可看完整输出;
5. 「停 止」随时中断;失败/中断的任务再次点「开始处理」会自动续跑。

## 结果浏览

- 左侧列出输出目录下所有处理过的视频,图标含义:📒 已生成讲义稿 / ✏️ 已纠错 / 🎙 仅转写 / 🖼 仅切页;
- 右侧内置 Markdown 预览(包含每页 PPT 截图);
- 「打开文件夹」直达输出目录;「导出 PDF」用本机 Chrome/Edge 渲染成 A4 PDF(可单个或全部导出)。

## 输出文件

每个视频在输出目录下生成独立文件夹:

| 文件 | 内容 |
| --- | --- |
| `slides/` | 每页 PPT 截图 |
| `slides.md` | 切页时间轴 |
| `slides_asr.md` | 原始逐字稿 |
| `slides_optimized.md` | 纠错后的逐字稿 |
| `slides_lecture_notes.md` | **讲义稿(最终产物)** |
| `asr.json` / `optimization.json` / `lecture_notes.json` | 各阶段过程数据 |
| `batch.log` | 运行日志 |

## 常见问题

**选了「完整讲义」却失败,提示 faster-whisper?** 本机转写引擎没装,去设置页点「安装」,或改用「全 API」方案。

**API 报 429 / 频繁重试?** 把设置里的「API 间隔」调大(如 10 秒)。任务会自动重试并续跑,不会丢进度。

**切页太碎或漏页?** 调「翻页灵敏度」:数值越小越敏感(切得越多)。动画渐显的 PPT 被合并属正常保护行为,可调小「防抖窗口」。

**只跑了转写,后悔没生成讲义?** 不用重跑转写:把方案换成「完整讲义」再点开始,已有的转写结果会被直接复用,只补跑纠错和讲义两步。

**转写太慢?** 设置里把 Whisper 模型换成 `tiny`/`base`,或推理设备改 `cuda`(需 NVIDIA 显卡);也可以直接用「全 API」。
